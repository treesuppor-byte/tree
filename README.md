### 7단계 가로 스크롤 강화 + 마우스 드래그 지원 🖱️📱

**CSS !important로 우선순위 확보 + 마우스 드래그로 데스크톱 테스트 가능**

---

#### 🎯 핵심 개선사항

**CSS 우선순위 문제 해결**:
- ✅ 모든 가로 스크롤 스타일에 `!important` 추가
- ✅ 기본 CSS의 `position: fixed` 덮어쓰기
- ✅ `overflow: visible` → `overflow-x: auto` 강제 적용
- ✅ `position: absolute` → `position: relative` 강제 변경

**마우스 드래그 기능 추가**:
```javascript
// 데스크톱에서도 드래그로 스크롤 가능
stage.addEventListener('mousedown', (e) => {
  isDragging = true;
  mouseStartX = e.pageX;
  startScrollLeft = stage.scrollLeft;
  stage.style.cursor = 'grabbing';
});

stage.addEventListener('mousemove', (e) => {
  if (!isDragging) return;
  const walk = (x - mouseStartX) * 2;
  stage.scrollLeft = startScrollLeft - walk;
});
```

**CSS 강화 내용**:
```css
@media (max-width: 1024px) {
  .circle-flow-wrapper {
    height: auto !important; /* 800vh 덮어쓰기 */
  }
  
  .circle-flow-container {
    position: relative !important; /* fixed 덮어쓰기 */
    height: auto !important;
  }
  
  .circle-flow-stage {
    position: relative !important; /* absolute 덮어쓰기 */
    display: flex !important;
    flex-direction: row !important;
    overflow-x: auto !important; /* visible 덮어쓰기 */
    overflow-y: hidden !important;
    scroll-snap-type: x mandatory !important;
  }
  
  .flow-card {
    position: relative !important; /* absolute 덮어쓰기 */
    transform: scale(1) !important;
    scroll-snap-align: center !important;
  }
}
```

**디버깅 로그 추가**:
```javascript
console.log('CircleFlow 모드 선택:', {
  width: window.innerWidth,
  mode: isSmallScreen ? '가로 스크롤' : '세로 스크롤'
});

  display: getComputedStyle(stage).display,
  flexDirection: getComputedStyle(stage).flexDirection,
  overflowX: getComputedStyle(stage).overflowX
});
```

- ✅ 1025px 이상: 기존 세로 스크롤 방식 (대형 PC)
- ✅ 오른쪽→왼쪽 스와이프로 다음 카드 이동

**터치 스와이프 로직 구현**:

**1. 스와이프 감지 시스템**:
```javascript
// 터치 시작
stage.addEventListener('touchstart', (e) => {
  touchStartX = e.touches[0].clientX;
});

// 터치 종료 - 방향 감지
stage.addEventListener('touchend', () => {
  const swipeDistance = touchStartX - touchEndX;
  
  if (swipeDistance > 50) {
    // 오른쪽→왼쪽: 다음 카드
    scrollToCard(currentStep);
  } else if (swipeDistance < -50) {
    // 왼쪽→오른쪽: 이전 카드
    scrollToCard(currentStep - 2);
  }
});
```

**2. 부드러운 카드 전환**:
```javascript
scrollToCard(index) {
  const card = cards[index];
  card.scrollIntoView({
    behavior: 'smooth',
    inline: 'center'
  });
}
```

**3. 반응형 크기 조정**:
```css
/* 1024px 이하 (태블릿) */
.flow-card {
  width: min(420px, 85vw);
  padding: 0 15px;
}

/* 768px 이하 (모바일) */
.flow-card {
  width: min(340px, 85vw);
  padding: 0 10px;
}
```

**4. 화면 크기 변경 감지**:
```javascript
handleResize() {
  const wasSmallScreen = lastWidth <= 1024;
  const nowSmallScreen = currentWidth <= 1024;
  
  // 1024px 기준 전환 시 새로고침
  if (wasSmallScreen !== nowSmallScreen) {
    window.location.reload();
  }
}
```

### SVG 아이콘 크기 충돌 문제 해결

**문제점**:
- 동일한 HTML 구조의 SVG 아이콘(width="40" height="40")이 특정 섹션에서만 작게 표시됨
- CSS 선택자 충돌: `.step-icon svg` 스타일이 여러 섹션에 걸쳐 적용되어 의도하지 않은 크기 변경 발생

**해결 방법**:
- ✅ CSS 선택자 구체화: `.step-icon svg` → `.flow-card .step-icon svg`
- ✅ 7단계 원형 흐름 섹션의 아이콘만 70px x 70px로 지정
- ✅ 다른 섹션의 `.step-icon`과 CSS 충돌 제거
- ✅ HTML의 width/height 속성이 정상적으로 적용되도록 수정

**변경된 CSS 선택자**:
```css
/* Before (충돌 발생) */
.step-icon svg { width: 70px; height: 70px; }

/* After (충돌 해결) */
.flow-card .step-icon svg { width: 70px; height: 70px; }
```

**검증 방법**:
- 개발자도구 → Computed 탭에서 모든 `.step-icon svg`의 width/height 확인
- 7단계 원형 흐름: 70px x 70px (데스크탑), 44px x 44px (모바일)
- 기타 섹션: HTML 속성대로 40px x 40px 정상 렌더링

#### ✅ 모바일 (768px 이하)
**"빠르고 투명한 3단계" 섹션과 완전히 동일한 방식**:

1. **구조**
   ```
   ┌─────────────┐
   │   카드 1    │ ← opacity: 0.4, scale: 0.92
   ├─────────────┤
   │   카드 2    │ ← active (보일 때)
   ├─────────────┤

   ├─────────────┤

   세로로 쭉 나열 (gap: 40px)
   페이지 전체 스크롤만 사용
   ```

2. **IntersectionObserver 방식**
   - threshold: 0.5 (50% 보일 때)
   - 화면에 보이면 → `.active` 클래스 추가
   - 화면에서 벗어나면 → `.dimmed` 클래스 추가
   - **3단계 섹션과 완전히 동일한 로직**

3. **CSS 클래스**
   ```css
   /* 기본 상태: 흐리게 */
   .flow-card {
       opacity: 0.4;
       transform: scale(0.92);
   }
   
   /* 활성 상태: 선명하게 */
   .flow-card.active {
       transform: translateY(-12px) scale(1.05) !important;
       opacity: 1 !important;
   }
   
   /* 비활성 상태 */
   .flow-card.dimmed {
       transform: scale(0.92);
       opacity: 0.4;
   }
   ```

4. **JavaScript 구현**
   ```javascript
   // 모바일: IntersectionObserver (3단계와 동일)
   initMobileObserver() {
       const options = {
           threshold: 0.5,
           rootMargin: '0px'
       };
       
       const observer = new IntersectionObserver((entries) => {
           entries.forEach(entry => {
               if (entry.isIntersecting) {
                   entry.target.classList.add('active');
                   entry.target.classList.remove('dimmed');
               } else {
                   entry.target.classList.remove('active');
                   entry.target.classList.add('dimmed');
               }
           });
       }, options);
       
       this.cards.forEach(card => observer.observe(card));
   }
   ```

   - ❌ 실시간 스타일 변경

**부드러운 관성 스크롤 시스템** (naver-seo.com 스타일):

**핵심 원리**:
```javascript
class SmoothScroll {
    // 1. Wheel 이벤트 가로채기
    onWheel(e) {
        e.preventDefault(); // 브라우저 기본 스크롤 차단
        this.targetScroll += e.deltaY * 1.2; // 목표 위치 업데이트
    }
    
    // 2. Easing으로 부드럽게 이동
    updateScroll() {
        const diff = this.targetScroll - this.currentScroll;
        this.currentScroll += diff * 0.08; // lerp (감속)
        window.scrollTo(0, this.currentScroll);
    }
    
    // 3. 섹션 스냅 효과
    applySnapEffect() {
        // 가까운 섹션에 자연스럽게 당김
        const pullStrength = (1 - distance / threshold) * 0.15;
        this.targetScroll += diff * pullStrength;
    }
    
    // 4. requestAnimationFrame
    raf() {
        this.updateScroll();
        requestAnimationFrame(() => this.raf());
    }
}
```

**작동 방식**:
```
사용자 휠 스크롤
    ↓
Wheel 이벤트 감지 (preventDefault)
    ↓
targetScroll 업데이트
    ↓
currentScroll이 targetScroll을 향해 천천히 이동 (ease 0.08)
    ↓
섹션 근처면 자연스럽게 당김 (snap effect)
    ↓
requestAnimationFrame으로 부드럽게 반복
    ↓
고급스러운 감속 효과 ✨
```

**스크롤 경험**:
- 천천히 스크롤: 화면이 붙잡힌 듯 안정적
- 빠르게 스크롤: 속도는 줄어들되 흐름 유지
- 섹션 근처: 자연스러운 감속 + 부드러운 정지
- 멈춤 없이: 항상 부드럽게 이어짐

**카드 애니메이션 로직**:
```javascript
updateCardAnimations(stepFloat) {
    cards.forEach((card, index) => {
        const cardStep = index + 1;
        const distance = cardStep - stepFloat;
        
        if (Math.abs(distance) < 0.01) {
            // 현재 활성 카드 (중앙)
            transform = 'translate(-50%, -50%) scale(1)';
            opacity = 1;
        } else if (distance > 0 && distance <= 1.5) {
            // 다음 카드 (오른쪽 → 중앙)
            const t = 1 - distance; // 0 → 1
            const x = -20 + ((-50 - (-20)) * t); // -20% → -50%
            const y = 10 + ((-50 - 10) * t);     // 10% → -50%
            const scale = 0.75 + (0.25 * t);     // 0.75 → 1
            opacity = 0.4 + (0.6 * t);           // 0.4 → 1
        } else if (distance < 0 && distance >= -1.5) {
            // 이전 카드 (중앙 → 왼쪽 아래)
            const t = Math.abs(distance); // 0 → 1
            const x = -50 + ((-80 - (-50)) * t); // -50% → -80%
            const y = -50 + ((-90 - (-50)) * t); // -50% → -90%
            const scale = 1 + ((0.6 - 1) * t);   // 1 → 0.6
            opacity = 1 + ((0.2 - 1) * t);       // 1 → 0.2
        }
        
        // 직접 스타일 적용 (CSS transition 활용)
        card.style.transform = transform;
        card.style.opacity = opacity;
    });
}
```

   ```
   - 화살표: 파란색 + 살짝 이동
   - 숫자: 색상 반전
   ```

## 🐛 버그 수정 (2026-02-09 최종)
